Kiến trúc đề xuất cho ActiveEdu

Tầng 1 — Đánh giá năng lực: IRT 1PL (đơn giản, đủ dữ liệu hiện có)

Tầng 2 — Theo dõi kiến thức: BKT per-article (mỗi bài = 1 KC)

Tầng 3 — Chọn nội dung: ZPD-based + interest signal từ click logs

Tầng 4 — Tương tác: Claude API với student profile context

Lưu trữ: NocoDB (articles) + localStorage (student state) + event log
